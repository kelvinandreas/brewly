## Atomic commits

One logical change = one commit. Examples:

- ✓ `feat(auth): add owner self-registration` — all files for registration
- ✓ `fix(orders): prevent duplicate item when retrying POST` — one fix, one commit
- ✗ `feat: add auth AND menu AND tables` — too much, split into 3 commits

## Format

```
<type>(<scope>): <subject>

<optional body>
<optional footer>
```

- `<type>`: `feat`, `fix`, `chore`, `docs`, `refactor`, `test`, `style`.
- `<scope>`: small, conventional — `auth`, `menu`, `tables`, `orders`, `kds`, `songs`, `reports`, `db`, `ci`, `docker`.
- `<subject>`: imperative present-tense, no period, lowercase first letter, ≤ 72 chars.

## Examples

```
feat(auth): add owner self-registration on empty database
fix(orders): prevent transition from completed to preparing
refactor(menu): extract price formatting into pkg/money
chore(deps): bump chi to v5.1.0
docs(adr): add ADR-005 explaining SSE over WebSocket
test(usecase): cover refresh token rotation edge cases
```

## Body (when needed)

Use the body to explain **why**, not what:

```
fix(songs): reject 4th request from same table token

Counting song_requests by token_jti was off-by-one because the row
was inserted before the count check. Re-ordered the transaction so
the rate limit is enforced before the insert.
```

## DO

- Imperative: "add", "fix", "remove", not "added", "fixes".
- One logical change per commit.
- Scope when it clarifies (most commits have one).

## DON'T

- No mention of Claude, AI, Copilot, "generated by", or any AI tool. Anywhere — message, body, footer, git config.
- No emoji unless the user explicitly asks.
- No "WIP", "stuff", "misc fixes" — split the work.
- No past tense ("fixed bug").
- No issue tracker leaks unless the project has one.
